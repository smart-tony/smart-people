# 晨报推送工作台部署交接清单

## 目标

把 `weekly-push-tool` 部署成一个可给非技术人员使用的网页/本地工具。用户只需要打开页面，完成：

1. 一键获取内容
2. 查看候选池
3. 勾选并导入条目
4. 编辑周报
5. 导出 HTML 或复制企微推送 JSON

## 推荐方案

### 方案 A：网页服务版

适合多人使用。需要部署人员准备：

- Linux 服务器或容器托管平台
- Python 3.11+
- Playwright Chromium 运行环境
- 域名和 HTTPS
- LLM API Key
- 基础账号密码保护

启动方式：

```bash
cd /path/to/weekly-push-tool
pip install -r backend/requirements.txt
python -m playwright install chromium
python backend/server.py
```

访问地址：

```text
http://服务器IP:8000
```

生产环境建议用 Caddy / Nginx 反向代理到 `127.0.0.1:8000`。

### 方案 B：Windows 本地程序版

适合给单个不懂技术的人使用。部署人员需要把项目打包成：

```text
晨报推送工具/
  启动晨报工具.exe
  app/
  backend/
  config/
  data/
  logs/
```

启动器负责：

- 启动 FastAPI
- 检查 `http://localhost:8000/api/health`
- 自动打开浏览器
- 后台写日志

### 方案 C：Cloudflare 临时网页发布

如果没有服务器，可以只把“HTML 发布成 3 天网页”放到 Cloudflare Worker + R2：

- Worker 提供 `POST /api/publish`
- R2 保存 HTML
- `GET /p/{id}` 返回 HTML
- R2 生命周期规则设置 3 天后删除

本项目后续可加“发布到网页”按钮，把生成的 HTML 上传到 Worker。

## 必填配置

`config/llm.config.json` 或环境变量：

```text
LLM_API_KEY=...
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat
```

`config/sources.config.json`：

- 确认每个模块至少有一个 `enabled: true` 的来源
- 不想让用户看到空候选池时，先禁用不稳定来源

## 验收流程

部署人员必须逐项确认：

1. 打开首页，标题显示“晨报推送工作台”
2. `/api/health` 返回 `status: ok`
3. 点击“来源管理”，能看到抓取模块、每来源抓取条数、候选池
4. 没有启用来源的模块，页面提示“当前模块没有启用来源”，不能直接崩
5. 点击“一键获取内容”，候选池显示标题、来源、原文链接、摘要、入选理由
6. 勾选候选后，“已选”数量正确变化
7. 点击“导入已选 → 条目”，跳到“周报条目”，并出现导入内容
8. “新增条目”按钮在顶部和周报条目页内都能用
9. “复制推送 JSON”能复制，失败时会弹出可手动复制内容
10. “导出 HTML”能下载单文件 HTML
11. 桌面宽度和手机宽度下页面没有横向滚动

## 已知注意事项

- Playwright 抓取部分站点可能被 CDN/WAF 拦截，这属于来源稳定性问题，不是页面渲染问题。
- 如果页面一直显示旧版样式，请清浏览器缓存或更新 `index.html` 里的 `app.js?v=` / `styles.css?v=`。
- LLM API Key 不建议写死进镜像，生产环境最好使用环境变量。
