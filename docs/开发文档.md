## 一、项目概述

**运小星日报**是百运网（BY56.COM）的跨境物流 + AI 技术资讯聚合平台。每日自动抓取多个权威信息源，提取核心内容，通过 SQLite 数据库持久化存储，前端以日报形式呈现。

### 核心能力

| 能力 | 说明 |
|------|------|
| 物流信息抓取 | 8+ 个专用爬虫，覆盖 WL123、出海网、雨果跨境、航运界等 |
| AI 技术聚合 | 对接 AI HOT API，获取每日 AI 精选 |
| LLM 分析（可选） | DeepSeek 做摘要/评分/标签，可关闭 |
| SQLite 持久化 | 自动去重、按日期查询、< 1ms 响应 |
| 前端日报 | 响应式设计，支持物流/AI 切换、日期导航、详情面板 |

## 四、数据库设计

### 表结构 `items`

```sql
CREATE TABLE items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,              -- 文章标题
    source_name TEXT DEFAULT '',      -- 来源名称（如"WL123 物流资讯"）
    source_url TEXT UNIQUE,           -- 原文链接（唯一索引，自动去重）
    summary TEXT DEFAULT '',          -- AI 摘要 / 正文摘取（150-200字）
    body_text TEXT DEFAULT '',        -- 原文正文（全文搜索用）
    image TEXT DEFAULT '',            -- 封面图 URL (og:image)
    ai_score REAL DEFAULT 5.0,       -- AI 评分（0-10）
    ai_tags TEXT DEFAULT '[]',       -- JSON 数组：["关税","清关","国际空运"]
    task_type TEXT DEFAULT '',       -- 模块：logistics-daily/global-news/...
    status TEXT DEFAULT 'candidate', -- candidate / published / hidden
    scraped_at TEXT NOT NULL,        -- 抓取时间（ISO 8601）
    published_at TEXT DEFAULT ''     -- 发布时间
);
```

### 索引

```sql
CREATE INDEX idx_status ON items(status);
CREATE INDEX idx_task ON items(task_type);
CREATE INDEX idx_score ON items(ai_score DESC);
CREATE INDEX idx_scraped ON items(scraped_at);
CREATE INDEX idx_source ON items(source_url);
```

### 特性

- **WAL 模式**：支持读写并发，多人查看不阻塞
- **URL 唯一索引**：`INSERT OR IGNORE` 自动去重
- **自动迁移**：新字段通过 `ALTER TABLE` 增量添加，不破坏旧数据

## 五、专用爬虫架构

### 设计原则

> **不用通用爬虫。** 每个网站 HTML 结构不同，必须精确匹配文章 URL 模式。

### 提取器注册表

```python
_EXTRACTORS = {
    "wl123.com":      _extract_wl123,       # /wu-liu-zi-xun/详情
    "chwang.com":     _extract_chwang,       # /news/数字
    "cifnews.com":    _extract_cifnews,      # /article/ /news/
    "ship.sh":        _extract_ship_sh,      # /articles/
    "egainnews.com":  _extract_egainnews,    # /article/数字
    "by56.com":       _extract_by56,         # /news/数字.html
    "kjdsnews.com":   _extract_kjdsnews,     # /a/数字.html
    "mofcom.gov.cn":  _extract_mofcom,       # 商务部关键词过滤
    "5688.cn":        _extract_5688,         # 物流巴巴
}
```

### 提取器模板

```python
def _extract_xxx(soup, base_url, max_items):
    """XX网站：只取文章页，过滤导航/服务/产品"""
    return _extract_by_url_patterns(
        soup, base_url, max_items,
        allowed_patterns=[r"/article/\d+"],
        blocked_patterns=[r"/service", r"/login"],
    )
```

### 新增网站步骤

1. 打开网站 → 找到文章列表页 URL
2. 打开一篇文章 → 观察 URL 模式
3. 写 `_extract_xxx()` 函数
4. 注册到 `_EXTRACTORS` 字典
5. 在 `config/sources.config.json` 添加源配置

## 六、数据流

### 写入流程（抓取 → 入库）

```
POST /api/admin/scrape-all
  │
  ├─ for 每个模块（物流资讯/国际风险/官方政策/跨境平台/航运港口/百运百科）
  │   │
  │   ├─ ① pick_sources(task_type)      → 从 config 拿到该模块的网站列表
  │   ├─ ② fetch_raw_articles(sources)   → 并发抓取列表页（每站调各自提取器）
  │   ├─ ③ deduplicate_by_url()          → URL 去重
  │   ├─ ④ enrich_article_content()      → 逐篇打开原文，提取正文+og:image，并清理客服/公众号/返回顶部等模板噪音
  │   ├─ ⑤ filter_recent_articles(days)  → 日期过滤（百运百科2天，其他5天）
  │   ├─ ⑥ analyze_with_llm()           → DeepSeek 摘要+打分（可选）
  │   └─ ⑦ filter_and_sort(threshold)    → 低分过滤（百运百科不过滤，其他≥5分）
  │
  └─ ⑧ insert_items()  → INSERT INTO items (自动跳过重复 URL)
```

### 读取流程（前端请求）

```
GET /api/briefing/logistics?date=2026-06-25
  │
  ├─ SELECT * FROM items
  │   WHERE status = 'published'
  │   AND scraped_at LIKE '2026-06-25%'
  │   ORDER BY ai_score DESC
  │   LIMIT 200
  │
  └─ 返回 JSON → 前端渲染（< 1ms）
```

### Fallback 逻辑

当天数据库无数据 → 自动触发实时抓取 → 写入数据库 → 返回结果

### 内容质量清洗

抓取层和读取层都会清理常见站点模板噪音，例如“客服”“加我微信”“公众号”“微信扫一扫关注”“返回顶部”“上一篇/下一篇/相关阅读”等。历史数据库中已经污染的摘要，会在 `/api/briefing/logistics` 返回前再次清洗；清洗后内容过短或与标题高度重复的条目会被过滤，避免把站点模板当成行业动态或政策风险展示。

补充规则：

- `ikjzd.com` 使用专用正文提取：优先读取 `.articlecontent`，兜底读取 `meta description`，不再使用客服浮层或公众号模板作为正文。
- 行业/政策动态不生成新增业务解读，能抓到正文就直接展示正文内容。
- 旧污染数据不追溯批量修复；从新抓取开始，正文提取和清洗规则生效。
- 当前验证口径：跨境知道模板、联系方式模板、导航面包屑、上一篇/下一篇推荐在 `/api/briefing/logistics` 输出中应为 0。

## 十、内容质量控制

### 后端过滤链

1. **URL 模式过滤**：专用提取器只抓匹配的文章 URL
2. **标题去噪**：去掉"N秒前"、"分享至"等干扰文字
3. **日期过滤**：只保留时效内的文章
4. **摘要去噪**：去掉"用小程序打开更快"、"打开APP"等网页噪音
5. **标题≈摘要检测**：摘要和标题字符重合 > 85% 则丢弃
6. **摘要最小长度**：< 30 字视为无实质内容
7. **标题精简**：超过 60 字自动截断，多余部分移入摘要
8. **摘要限制**：200 字，在句号/逗号处自然截断

### 前端展示限制

- 摘要：CSS `line-clamp: 3`（最多 3 行）
- 标签：最多 4 个，过长(>15字)的过滤
- 封面图：通过 `/api/img-proxy` 代理（防微信盗链）

## 十一、部署与运维

### 启动

```bash
cd /opt/weekly-push-tool/backend
uvicorn server:app --host 0.0.0.0 --port 8000
```

### 手机端本地预览

本机调试时，如果只监听 `127.0.0.1`，手机无法访问。需要监听局域网地址：

```bash
python3 -m uvicorn backend.server:app --host 0.0.0.0 --port 8000
```

手机与电脑连接同一 Wi-Fi 后，用电脑局域网 IP 访问：

```
http://<电脑局域网IP>:8000/daily
```

例如本机当前检测到的局域网地址是 `192.168.1.41`，手机可访问：

```
http://192.168.1.41:8000/daily
```

### 定时抓取

当前由 `server.py` 的 `_auto_refresh_loop` 内置执行，每 15 分钟自动抓取一次。
Docker 部署不再额外启动 crontab、sidecar 或 `cron_scraper.py --loop`，避免重复抓取。

```
/opt/weekly-push-tool/data/weeks.db
```
