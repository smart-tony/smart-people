## 一、项目概述

**weekly-push-tool** 是百运科技的内容推送工作台，把每周/每日内容推送从"手工改 HTML"升级为"傻瓜式填充 + 一键获取 + 自动排版 + 可发布 URL"。

### 使用对象

- 非技术同事、内容编辑、运营同学

### 核心场景

| 场景 | 说明 |
|------|------|
| AI 科技周报 | 固定网站采集 → LLM 整理 → 人工勾选 → 自动生成 HTML 和企微推送 |
| 每日物流快讯 | 战争/极端天气/港口/航线/清关变化等跨境物流风险信息 |
| 汇率快讯 | 美元/人民币/欧元等汇率变化及对报价的影响 |
| 天气预警 | 极端天气、台风路径等 |

| 其他 | `_extract_generic` | 所有 `<a>` + 噪音过滤 | 兜底方案 |

### Playwright 浏览器池

```
_BrowserPool（单例）
├── 懒初始化：首次使用时启动 Chromium
├── render_page(url)：创建独立 context + page → 渲染 → 关闭 context
├── atexit 注册 shutdown()：进程退出时关闭浏览器
└── Semaphore(2)：并发控制，最多 2 个同时渲染
```

### URL 缓存机制

```
格式：md5hex|timestamp（每行一条）
示例：a1b2c3d4e5f6...|1718700000.0

加载（_load_cache）：
  - 解析每行，有 | 的按时间戳判断是否过期（7天 TTL）
  - 无 | 的旧行视为永不过期
  - 过期条目跳过

写入（_save_to_cache）：
  - 新条目带时间戳追加
  - 文件超过 500 行时重写，去掉过期条目
```

#### scrape.py 6 处改动详情

1. **task_type 别名映射**（顶部新增）
   - 新增 `TASK_TYPE_ALIASES` 字典和 `resolve_task_type()` 函数
   - `fetch_pipeline` 中 `analyze_with_llm` 使用转换后的标准名

2. **kjdsnews 专用提取器**（_extract_generic 前新增）
   - 新增 `_extract_kjdsnews()` 函数，使用 `.title a` 选择器
   - 过滤 `/a/\d+\.html` 链接模式，最小标题长度 6 字符
   - 加入 `_EXTRACTORS` 字典

3. **LLM 失败条目不再加入候选池**（analyze_with_llm 函数）
   - 删除 `candidates.append(CandidateItem(...))` 的 fallback 逻辑
   - 失败信息只记录到 errors 列表，错误前缀加 ⚠️

4. **Playwright 浏览器复用池**（替换 _render_with_playwright）
   - 新增 `_BrowserPool` 单例类：懒初始化、context 隔离、atexit 清理
   - `_render_with_playwright` 改为调用 pool.render_page()

5. **test_source 代码去重**（test_source 函数）
   - 提取 `_try_playwright_fallback()` 公共函数
   - 4 段重复 Playwright fallback 代码简化为 1 处调用

6. **URL 缓存过期机制**（_load_cache / _save_to_cache 重写）
   - 缓存格式从纯 MD5 改为 `md5hex|timestamp`
   - 7 天 TTL，超 500 行自动清理过期条目
   - 向后兼容旧格式（无时间戳的行视为永不过期）
