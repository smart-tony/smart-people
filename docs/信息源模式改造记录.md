# 信息源模式改造记录

> 创建时间：2026-06-18
> 当前状态：方案设计阶段，尚未按本方案改代码

---

## 1. 本次方向

当前工具先不再以“自动生成完整周报模板”为核心，也不再优先做网页导出，而是改成“权威信息源获取 + 人工选择 + 图片辅助”的编辑素材池。

目标用户自己制作最终模板，我们只负责给编辑人提供准确、可追溯、可勾选的信息源、来源链接和图片素材。

### 新定位

```text
旧模式：抓取新闻 → AI 整理 → 导入周报 → 右侧 HTML 预览 → 导出完整周报

新模式：抓取权威信息源 → 形成信息池 → 用户勾选来源和图片 → 交给编辑人制作模板
```

---

## 2. 为什么要改

### 用户真实需求

- 对方会自己制作模板，不一定需要我们生成完整周报版式。
- 对方更需要“准确的信息来源”和“可直接引用的素材”。
- 当前右侧 HTML 看板偏向最终排版，对“信息源筛选”帮助不大。
- 如果模板由对方自己写，我们不应该在工具里强绑定固定周报样式。

### 现有问题

| 问题 | 影响 |
|---|---|
| 右侧是完整 HTML 预览 | 用户容易以为必须按我们的模板发布 |
| 候选池和周报编辑耦合太强 | 信息源筛选和最终写作混在一起 |
| 分区逻辑服务于周报排版 | 对“只要信息源”的场景过重 |
| 图片只是条目附属字段 | 不适合做素材选择和后续模板制作 |

---

## 3. 新工作流

### 推荐流程

1. 选择信息类型
2. 点击“获取今日信息源”
3. 系统从固定权威网站抓取信息
4. 系统生成“信息源池”
5. 用户查看标题、摘要、来源、原文链接、重要性
6. 用户勾选需要的信息源
7. 用户为每条信息选择或粘贴图片
8. 右侧生成“已选信息源清单”
9. 编辑人根据已选信息源、来源链接和图片自行制作最终模板

## 4. 页面框架调整

### 左侧：信息获取与筛选

左侧保留为主要操作区，但名称和功能需要调整。

| 区域 | 新名称 | 功能 |
|---|---|---|
| 基础信息 | 项目信息 | 标题、日期、用途说明 |
| 来源管理 | 信息源管理 | 固定权威来源开关、测试来源 |
| 候选池 | 信息源池 | 抓取后的候选信息 |
| 周报条目 | 已选信息 | 用户勾选后的信息素材 |
| 企微推送 | 可后置或隐藏 | 当前阶段不是核心 |

### 右侧：不再是 HTML 看板

右侧从“HTML 预览”改成“已选信息源面板”。

右侧应展示：

- 已选信息数量
- 按类别分组的信息源
- 每条信息的来源、原文链接、摘要
- 图片选择状态
- 当前可交给编辑人的素材清单

### 右侧不再做什么

- 不再展示完整周报模板。
- 不再强制显示固定结语、期号、企微卡片样式。
- 不再把所有内容塞进统一新闻卡片模板。

## 5. 信息分类

当前建议保留 4 类：

| 类别 | 用途 |
|---|---|
| 关税政策 | 关税调整、贸易政策、清关政策 |
| 地缘风险 | 航运安全、海峡风险、地区安全事件 |
| 石油能源 | 油价、燃油、能源供应、库存 |
| 汇率数据 | 常见货币汇率和较上一交易日浮动 |

### 分类原则

优先级：

1. 来源配置中的分类前缀，例如 `关税政策｜USTR`
2. 用户手动选择的分区
3. 关键词兜底

不再使用“综合动态”栏目。

```json

```

## 9. 需要牺牲/暂停的功能

本版本为了让工具变轻，需要暂时牺牲一些旧功能。

| 暂停内容 | 原因 |
|---|---|
| 右侧完整 HTML 周报预览 | 不再由工具决定最终模板 |
| 固定周报结语、期号、卡片式版式 | 对信息源场景不是核心 |
| 企微 JSON 优先级降低 | 当前先解决信息准确性和素材筛选 |
| 自动把候选导入周报正文 | 改为先形成已选信息源清单 |
| 过度依赖 AI 分类 | 分类优先由来源决定，AI 只做摘要和重要性 |

## 10. 改造步骤

### Step 1：文档先行

状态：已开始

动作：

- 新增本文件，记录信息源模式的产品方向。
- 明确不再以完整周报模板为第一目标。
- 明确右侧改为信息源选择面板。

### Step 2：前端命名调整

状态：已做第一版

计划：

- `一键获取内容` 改为 `获取今日信息源`
- `候选池` 改为 `信息源池`
- `周报条目` 改为 `已选信息`
- `HTML 预览` 改为 `已选信息源`

牺牲：

- 用户短期内会看到旧字段和新字段混用，需要一次性改干净。

实际改动：

- 页面标题改为“信息源素材工作台”。
- 顶部“导出 HTML”入口移除。
- 来源抓取按钮改为“获取今日信息源”。
- 候选区文案改为“信息源池”。
- 左侧已选内容改为“已选信息”。
- 交付页改为复制素材 Markdown / JSON。

### Step 3：右侧看板重构

状态：已做第一版

计划：

- 移除 iframe HTML 预览。
- 右侧改成普通 DOM 面板。
- 展示已选信息源、图片状态和导出按钮。

牺牲：

- 暂时不展示最终周报长图/长网页效果。

实际改动：

- 右侧 iframe 预览从页面移除。
- 右侧改为“编辑素材篮”。
- 已选信息按分类分组展示。

### Step 4：已选信息数据结构

状态：待做

### Step 5：交付信息源素材

- 生成“信息源素材清单”，不是完整周报模板，也不是网页导出主流程。

- 给编辑人复制 Markdown 或下载 JSON，方便他们自己制作模板。
- 汇率作为独立表格保留。

牺牲：

- 不再自动套用原来的漂亮周报样式。
- 不再把“网页导出”作为核心功能。

## 13. 2026-06-18 需求修正：不做网页导出

用户确认：编辑人会自己制作模板，工具不需要负责生成网页版本。

### 新交付物

工具只需要给编辑人一份可用素材：

- 合理的信息源
- 来源名称
- 原文链接
- 简短摘要
- 为什么值得关注
- 所属类别
- 可选图片或图片链接
- 汇率数据及浮动

### 右侧面板新目标

右侧不是网页预览，也不是 HTML 看板，而是“编辑素材篮”。

编辑素材篮应该支持：

- 查看已勾选信息
- 修改类别
- 粘贴或选择图片
- 删除不需要的信息
- 复制给编辑人的素材清单
- 下载 JSON / Markdown，供后续人工制作模板

### 后续可扩展

用户后续会补充相关网页文档。收到新文档后，需要：

1. 先记录进本文件。
2. 判断它属于哪类来源。
3. 判断是官方来源、媒体来源还是内部资料。
4. 再决定是否加入默认信息源库。

### I. 2026-06-21 LLM 余额不足兜底

问题：

- 抓取成功后，每条信息会调用 LLM 做摘要和打分。
- 当模型接口返回 `402 Insufficient Balance` 时，之前每条都会报错，且失败条目不会进入候选池。
- 在“信息源素材池”模式下，这不合理，因为即使没有 AI 摘要，原始标题、来源链接也仍然有价值。

改动：

- 后端在 LLM 失败时，仍生成一个原始信息源候选。
- 该候选保留标题、来源名称、原文链接、正文片段。
- 标签标记为 `未AI分析`。
- 摘要提示“LLM 暂不可用，已保留原始标题和来源链接”。
- 前端把大量 `Insufficient Balance / 402` 错误压缩成一条提示，避免刷屏。

牺牲：

- 没有余额时，候选的摘要质量下降。
- AI 分数统一兜底为 5 分，不代表真实重要性。

收益：

- LLM 没钱或接口异常时，信息源池仍然可用。
- 编辑人仍可以先人工筛选来源，不会因为模型失败导致素材池为空。

### P. 2026-06-23 Playwright 线程稳定性修复

问题：

- AI 科技周报抓取多个 JS 站点时，出现：
  - `Cannot switch to a different thread`
  - OpenAI `Page.goto timeout`
- 原因是后端复用了一个全局 `sync_playwright` 浏览器池，但抓取接口会在线程池中并发调用 Playwright。
- `sync_playwright` 创建在哪个线程，就只能在哪个线程使用，跨线程复用会触发 greenlet 线程切换错误。

改动：

- 移除全局 Playwright 浏览器池。
- Playwright 渲染改为官方 async API，不再在线程池中调用 `sync_playwright`。
- 每次 Playwright 渲染都独立启动和关闭 Chromium。
- `page.goto` 等待条件从 `domcontentloaded` 调整为 `commit`，再短暂等待 DOM/networkidle，减少慢站点直接超时。

取舍：

- 抓取 JS 站点会比复用浏览器略慢。
- 但不会因为一个站点的 Playwright 线程错误拖垮其他来源，稳定性更适合当前编辑工具。
